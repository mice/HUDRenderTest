## 1. 架构概述

HUDRenderTest 采用**分层管线架构**，将 HUD UI 元素的生成、内存管理、数据合并、异步调度和最终渲染解耦为独立模块。整体目标是通过将多个 UI 元素合并为单次 `Graphics.DrawMesh()` 调用来最小化 Draw Call 数量。

**核心架构特征：**

| 特征 | 说明 |
|------|------|
| 接口驱动 | 通过 `IUIDrawTarget`、`IUIData`、`IUIPrefabHolder` 接口解耦各层 |
| 内存池化 | 自定义 `UIGeometry` 内存池避免 GC 分配 |
| 双模式数据容器 | `UIMeshData`（独立数组）与 `UIMeshDataX`（共享池）两种策略 |
| 异步管线 | 通过 Job System 桥接层将 Mesh 合并移出主线程 |
| 多纹理批处理 | 单次 Draw Call 支持最多 8 张纹理（UV.z 通道索引） |

---

## 2. 设计原则

| 原则 | 体现 |
|------|------|
| **单一职责** | 每个模块仅负责一项核心功能（内存分配、数据填充、纹理管理等） |
| **接口隔离** | 组件层仅依赖 `IUIData` 接口，不关心底层是独立数组还是共享池 |
| **零 GC 分配** | 运行时通过内存池复用，避免每帧 new/alloc |
| **数据与行为分离** | Mesh 数据（`UIMeshData`/`UIMeshDataX`）与 UI 组件行为（`UIText`/`UIImage`）独立 |
| **可替换渲染策略** | 通过切换 `UseSlim(bool)` 可在两种数据容器间动态切换 |
| **异步友好** | Job System 集成设计允许主线程与工作线程并行处理 |

## 4. 模块详细设计

### 4.1 核心接口层（Core）

**文件：** `Assets/Scripts/UIDataRender/Core.cs`

**职责：** 定义系统核心抽象，解耦上下游模块。

```
┌──────────────────────────┐
│      IUIDrawTarget       │  UI 组件必须实现此接口
│──────────────────────────│
│ + DoGenerate(IUIData,    │  将组件顶点数据填充至数据容器
│              Transform)  │
└──────────────────────────┘

┌──────────────────────────┐
│         IUIData          │  Mesh 数据容器必须实现此接口
│──────────────────────────│
│ + TextureIndex           │  纹理槽索引（1～7）
│ + FillVertex(...)        │  从 VertexHelper 填充
│ + UpdateTextureIndex(...)│  更新 UV.z 通道
│ + TransformVertex(...)   │  矩阵变换顶点
│ + FillToDrawData(...)    │  追加至合并缓冲
│ + FillToTriangleData(...)│  追加索引数据
└──────────────────────────┘

┌──────────────────────────┐
│     IUIPrefabHolder      │  Prefab 实例持有者接口
│──────────────────────────│
│ + BuildMesh(...)         │  初始化 Mesh 数据
│ + Fill(...)              │  填充渲染缓冲
│ + SetText/Sprite/Width   │  动态更新
└──────────────────────────┘
```

**设计要点：**
- 所有 UI 组件（`UIText`、`UIImage`）通过 `IUIDrawTarget` 统一调度
- 数据容器通过 `IUIData` 接口使上层无需关心底层存储策略
- 预制体持有者通过 `IUIPrefabHolder` 支持多种实现

---

### 4.2 几何内存池（UIGeometry）

**文件：** `Assets/Scripts/UIDataRender/UIGeometry.cs`

**职责：** 管理全局共享的顶点/索引缓冲区，实现零 GC 的内存分配策略。

```
┌───────────────────────────────────────────────────────┐
│                     UIGeometry                         │
│───────────────────────────────────────────────────────│
│ - vertex: Vector3[]        (初始 4096)                │
│ - drawVertex: Vector3[]    (渲染用顶点副本)            │
│ - uvs: Vector4[]           (XY=UV, Z=纹理索引, W=flags)│
│ - colors: Color32[]        (顶点颜色)                  │
│ - indices: int[]           (初始 8192)                 │
│ - openVertexList: LinkedList<MeshSlim>  (空闲顶点段)   │
│ - openIndicesList: LinkedList<MeshSlim> (空闲索引段)   │
│───────────────────────────────────────────────────────│
│ + Alloc(vertexCount, indiceCount): MeshSlim           │
│ + Release(mesh: MeshSlim): void                       │
│ + ReAlloc(vertexCount, indiceCount, ref mesh): void   │
│ + ReleaseSlice(mesh: MeshSlim): void                  │
└───────────────────────────────────────────────────────┘
```

**内存分配策略：**

```
分配请求 → 搜索空闲链表
            ├── 精确匹配 → 移除节点，返回
            ├── 更大段 → 拆分，前半返回，后半留链表
            └── 无匹配 → 扩容数组（步长 GROWTH=4096），添加新空闲段
```

**释放合并策略（5 种边界情况）：**

```
Case 1: [释放块] ... [链表头]     → 独立插入链表前端
Case 2: [释放块][链表头]          → 合并入链表头
Case 3: [前节点][释放块][后节点]  → 三段合一
Case 4: [前节点][释放块] ... [后] → 合并入前节点
Case 5: [前] ... [释放块][后节点] → 合并入后节点
```

### 4.3 Mesh 数据容器层（MeshData）

**文件：** `Assets/Scripts/UIDataRender/UIMeshData.cs`、`UIMeshDataX.cs`

**职责：** 存储单个 UI 元素的几何数据，提供两种存储策略。

#### 策略对比

| 维度 | UIMeshData（独立数组） | UIMeshDataX（共享池） |
|------|------------------------|----------------------|
| 内存模型 | 每实例独立 `Vector3[]`/`Vector4[]`/`int[]` | 仅持有 `MeshSlim` 描述符，数据在 `UIGeometry` 中 |
| GC 压力 | 每次创建/销毁产生分配 | 零 GC（池化分配） |
| 适用场景 | 少量元素、需要独立变换 | 大量元素、内存敏感 |
| 矩阵变换 | 支持（`FillWithMatrix`） | 待实现 |
| Job 兼容 | 可通过 `SharedArray` 传递 | 直接操作共享缓冲区 |

#### MeshSlim 描述符

```
struct MeshSlim {
    int VertexOffset;    // 在共享数组中的顶点起始位置
    int VertexCount;     // 顶点数量
    int IndicesOffset;   // 在共享数组中的索引起始位置
    int IndicesCount;    // 索引数量
    int Index;           // 唯一标识（-1 = 无效）
}
```

### 4.4 UI 组件层（UI Components）

**文件：** `Assets/Scripts/UIDataRender/UI/`

**职责：** 从 Unity UGUI 组件中提取顶点数据，转换为自定义 Mesh 格式。

```
                    ┌─────────────────┐
                    │  IUIDrawTarget  │
                    └────────┬────────┘
                             │ implements
              ┌──────────────┼──────────────┐
              │              │              │
    ┌─────────▼──────┐  ┌───▼────────┐   ┌▼───────────┐
    │     UIText     │  │  UIImage   │   │ UIOutline  │
    │  (继承 Text)   │  │(继承 Image)│   │(继承Outline)│
    └─────────┬──────┘  └───┬────────┘   └┬───────────┘
              │              │              │ 修饰
              │              │              │
              └──────────────┼──────────────┘
                             │ DoGenerate
                             ▼
                    ┌─────────────────┐
                    │     IUIData     │
                    └─────────────────┘
```

**组件职责：**

- **UIText**：提取文字顶点/UV，纹理索引固定为 1（字体纹理），支持 UIOutline 描边
- **UIImage**：提取图片顶点/UV，纹理索引由 `UIPrefabManager` 动态分配
- **UIOutline**：修饰 UIText，在原始顶点基础上添加描边顶点

### 4.5 预制体管理层（Prefab Management）

**文件：** `Assets/Scripts/UIDataRender/UIPrefabManager.cs`、`UIPrefaHolder.cs`、`DataPrefabHolder.cs`、`UIPrefabOwner.cs`、`UIPrefabRegistration.cs`

**职责：** 管理 Prefab 实例的注册、生命周期、纹理状态和动态更新。

```
┌──────────────────────────────────────────────────────────────────┐
│                    UIPrefabManager（全局单例）                     │
│──────────────────────────────────────────────────────────────────│
│ - registrations: Dictionary<Prefab, UIPrefabRegistration>        │
│ - holders: List<IUIPrefabHolder>                                 │
│ - textures: List<Texture>                                        │
│──────────────────────────────────────────────────────────────────│
│ + Register(holder) → 创建 Registration                           │
│ + Generate(holder) → 触发 BuildMesh                              │
│ + GetTextureIndex(Texture) → int                                 │
│ + UpdateTexture(Material) → 同步 _MainTex1～7                    │
│ + OnTextureRegister/UnRegister → 纹理生命周期                     │
└──────────────────────────────────────────────────────────────────┘
         │                          │                    │
         │ 管理                     │ 注册               │ 同步纹理
         ▼                          ▼                    ▼
┌─────────────────┐    ┌───────────────────────┐   ┌──────────┐
│  UIPrefaHolder  │    │ UIPrefabRegistration  │   │ Material │
│ (MonoBehaviour) │    │ (Prefab → Draws 映射) │   │ Shader   │
└────────┬────────┘    └───────────────────────┘   └──────────┘
         │ 持有
         ▼
┌─────────────────────────┐
│  DataPrefabHolder<T>    │
│  T: UIMeshData          │
│     或 UIMeshDataX      │
└─────────────────────────┘
```

**纹理管理策略：**

```
注册纹理：
  textures.Add(newTexture) → 返回 index (1-based)

注销纹理（Swap-with-Last）：
  textures[removedIndex] = textures[lastIndex]
  textures.RemoveAt(lastIndex)
  通知所有 Holder 更新 TextureIndex
```

### 4.7 渲染层（Rendering）

**文件：** `Assets/Resources/Shader/UIE-AtlasBlit.shader`、`Assets/Art/Material/UIText.mat`

**职责：** 执行最终的 GPU 渲染，通过自定义 Shader 支持多纹理单次绘制。

```
┌─────────────────────────────────────────────────────────────────┐
│                  UIE-AtlasBlit Shader                             │
│─────────────────────────────────────────────────────────────────│
│  Properties:                                                     │
│    _MainTex0 (字体纹理, 槽 0)                                    │
│    _MainTex1 ~ _MainTex7 (图片纹理, 槽 1~7)                     │
│                                                                  │
│  Fragment Shader 逻辑:                                           │
│    texIndex = floor(uv.z)                                        │
│    color = tex2D(_MainTex[texIndex], uv.xy)                      │
│    output = color * vertexColor (Alpha Blend)                    │
└─────────────────────────────────────────────────────────────────┘
```

**渲染调用链：**

```
用户脚本 Update/LateUpdate
    │
    ├── 合并所有 UI 元素的顶点/UV/颜色/索引至 SharedArray
    │
    ├── 构建 Mesh（SetVertices, SetUVs, SetColors, SetTriangles）
    │
    ├── UIPrefabManager.UpdateTexture(material) → 设置 _MainTex1~7
    │
    └── Graphics.DrawMesh(mesh, Matrix4x4.identity, material, layer)
                → 单次 Draw Call 绘制所有 HUD 元素
```

## 5. 数据流与调用链

### 5.1 初始化流程

```
1. UIPrefabOwner 标记 Prefab 下的 UI 元素列表
            │
2. UIPrefaHolder.Start() → UIPrefabManager.Register(this)
            │
3. UIPrefabManager 创建 UIPrefabRegistration
   ├── 扫描所有 IUIDrawTarget 组件
   └── 为 UIImage 注册纹理 → textures 列表
            │
4. UIPrefabRegistration.Generate(holder)
   └── holder.BuildMesh(draws)
       └── DataPrefabHolder<T>.BuildMesh(draws)
           └── 对每个 draw: T data = new T(); draw.DoGenerate(data, root)
```

### 5.2 每帧渲染流程（非 Job 模式）

```
Update():
  1. 清空合并缓冲区（vertList, uvList, colorList, indexList）
  2. 遍历所有 Holder:
     └── holder.Fill(vertList, uvList, colorList, indexList, positions)
         └── DataPrefabHolder.Fill(...)
             └── 对每个 IUIData: data.FillToDrawData(...) 或 data.FillToTriangleData(...)
  3. mesh.SetVertices(vertList)
     mesh.SetUVs(uvList)
     mesh.SetColors(colorList)
     mesh.SetTriangles(indexList)
  4. UIPrefabManager.UpdateTexture(material)
  5. Graphics.DrawMesh(mesh, identity, material, layer)
```

### 5.3 每帧渲染流程（Job 模式）

```
Update():
  1. 准备 SharedArray 缓冲区
  2. 调度 Jobs:
     ├── MergeVertexJob.Schedule()   ─┐
     ├── MergeColorJob.Schedule()    ─┤ 并行
     └── MergeUVJob.Schedule()       ─┘
                                       │ 依赖
                                       ▼
     └── MergeIndicsJob.Schedule()     (等待上述完成)

LateUpdate():
  3. JobHandle.Complete() → 所有 Job 完成
  4. 从 SharedArray 中读取合并结果 → 设置 Mesh
  5. Graphics.DrawMesh(...)
```

### 5.4 动态更新流程

```
SetText(index, "新文字"):
  1. UIPrefaHolder.SetText(index, text)
  2. → DataPrefabHolder.SetText(index, text)
  3.   → UIText.text = text （触发 Unity Text 重绘）
  4.   → UIText.DoGenerate(meshData, root) （重新生成 Mesh 数据）
  5.     → meshData.FillVertex(vertexHelper, flags)
  6.       → UIGeometry.ReAlloc(...) （如果顶点数变化）

SetSprite(index, newSprite):
  1. UIPrefaHolder.SetSprite(index, sprite)
  2. → UIImage.sprite = newSprite
  3. → UIPrefabManager.OnTextureRegister(newTexture)
  4.   → 更新 textures 列表 & 通知 Material
  5. → UIImage.DoGenerate(meshData, root) （重新生成）
```

## 8. 关键设计决策

### 8.1 为什么使用自定义 Mesh 合并而非 UGUI Canvas 合批？

| 因素 | UGUI Canvas | 本框架 |
|------|-------------|--------|
| 动态内容 | 频繁 Rebuild 导致性能损失 | 仅更新变化部分的数据 |
| Draw Call | 受材质/层级/重叠影响，难以保证合批 | 强制合并为 1 次 Draw Call |
| 内存控制 | 由 Unity 内部管理，不可控 | 自定义内存池，零 GC |
| 多纹理 | 需要手动 Atlas 或多材质 | 单 Shader 支持 8 张纹理 |

### 8.2 为什么使用 ManagedCodeInJob 而非纯 Burst Job？

Unity Job System 要求 Job 数据为 unmanaged 类型，而 `UIMeshData` 持有托管数组（`Vector3[]` 等），无法直接放入 `NativeArray`。解决方案：

1. ~~方案一：将 UIMeshData 转为 NativeStruct~~ → 失败（`InvalidOperationException`）
2. **方案二（当前）：通过 `GCHandle` 固定托管对象，在 Job 中调用托管代码**
3. 方案三（未来）：将数据完全迁移至 NativeArray，使用 Burst 编译

### 8.3 双模式数据容器（UIMeshData vs UIMeshDataX）

```
场景决策树:

  元素数量少（< 100）且需要独立变换？
    → 使用 UIMeshData（独立数组模式）
    
  元素数量多（100+）且内存敏感？
    → 使用 UIMeshDataX（共享池模式）
    
  运行时通过 UseSlim(bool) 切换
```

### 8.4 纹理管理的 Swap-with-Last 策略

当某纹理被注销时，不直接删除（会导致数组空洞），而是将最后一个纹理交换至被删除位置：

```
删除前: [FontTex, TexA, TexB, TexC]  （索引 0,1,2,3）
删除 TexA (index=1):
  textures[1] = textures[3]  → [FontTex, TexC, TexB]
  通知所有使用 TexC 的 Holder: TextureIndex 从 3 → 1
```

此策略保持数组紧凑，避免遍历时跳过空位。

## 9. 扩展点与约束

### 9.1 扩展点

| 扩展点 | 方式 | 说明 |
|--------|------|------|
| 新 UI 组件类型 | 实现 `IUIDrawTarget` | 如 UIProgressBar、UIShape 等 |
| 新数据容器 | 实现 `IUIData` | 如基于 NativeArray 的纯非托管容器 |
| 新 Mesh 合并策略 | 实现 `IJobTask` | 如空间分区合并、LOD 合并等 |
| 纹理槽扩展 | 修改 Shader | 当前硬限制 8 张，可通过 Texture2DArray 扩展 |
| 新 Prefab 类型 | 实现 `IUIPrefabHolder` | 支持不同的 Prefab 管理策略 |

### 9.2 架构约束

| 约束 | 原因 | 影响范围 |
|------|------|----------|
| 最多 8 张纹理/Draw Call | Shader `_MainTex0~7` 硬编码 | 纹理管理层 |
| 不支持 Canvas Masking | 绕过 UGUI 裁剪系统 | UI 组件层 |
| Job 中不可使用 Burst | 托管代码通过 GCHandle 传递 | Job 桥接层 |
| `allowUnsafeCode: true` | `UnsafeFastCopy` 使用指针操作 | 程序集配置 |
| Unity 2021.x+ | 使用了 `VertexHelper` 扩展方法 | 全局 |

### 9.3 技术债务

| 项目 | 优先级 | 说明 |
|------|--------|------|
| `UIMeshDataX.FillToDrawData` 空实现 | P0 | 共享池模式无法正常渲染 |
| `using TreeEditor`/`UnityEditor` | P0 | Runtime Build 报错 |
| `UIPrefaHolder` 拼写错误 | P2 | 类名与文件名不一致 |
| 缺少 NativeArray 全链路 | P2 | 阻碍 Burst 优化 |
