## 0. 阅读与执行约定

每个测试用例包含 5 个字段：

| 字段 | 含义 |
|---|---|
| **简介** | 1 句话描述用例做什么 |
| **测试目标** | 验证哪条算法/契约（关联 [架构设计-v2](架构设计-v2.md) 章节 或 [需求分析](需求分析.md) 条目） |
| **测试预期** | 可断言的输出 / 状态 |
| **源文件** | NUnit 用例所在文件相对路径 |
| **被测代码** | 主要被验证的生产代码文件 |

**状态标记**：✅ 已存在　🆕 待新增　🛠 需补充

**通用前置**：
- EditMode 用例位于 `Assets/Tests/EditMode/*.cs`，PlayMode 位于 `Assets/Tests/PlayMode/*.cs`。
- 涉及 `UIMeshDataX` 的用例须在 `[SetUp]` 调用 `UIMeshDataX.geometry.Reset()`（待 [TASK-04](实施计划.md#task-04抽出-geometrybuffer--实现统一扩容) 提供）。
- 执行命令见 [实施计划.md §0](实施计划.md#0-执行说明给其他-agent)。

## 17. 测试编写规范（给 Agent）

1. **命名**：用例方法名采用 `场景_期望` 风格（示例：`Alloc_NoFit_TriggersGrow`）。
2. **断言**：优先 `Assert.AreEqual(expected, actual)`；浮点 `Assert.That(actual, Is.EqualTo(expected).Within(1e-4))`。
3. **日志断言**：用 `LogAssert.Expect(LogType.Warning, new Regex(...))`；用例末尾 `LogAssert.NoUnexpectedReceived()`。
4. **静态状态隔离**：`UIMeshDataX.geometry` 与 `UIPrefabManager.Instance` 需在 `[SetUp]` 重置（必要时反射）。
5. **MonoBehaviour fixture**：EditMode 中通过 `new GameObject("...").AddComponent<>()`，`[TearDown]` 用 `Object.DestroyImmediate`。
6. **PlayMode**：使用 `[UnityTest]` + `yield return null` 等待帧。
7. **数据等价比较**：双模式等价用例统一用 `CollectionAssert.AreEqual` 比较 list。
