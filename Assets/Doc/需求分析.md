## 1. 项目概述

**HUDRenderTest** 是一个基于 Unity 引擎的 **HUD（平视显示）UI 渲染性能优化框架**。

其核心思路是：将多个独立的 UI 元素（文字、图片）合并为一次 `Graphics.DrawMesh()` 调用，从而在游戏运行时极大减少 Draw Call 数量，提升渲染性能。

框架支持：
- 单次 Draw Call 中最多 **8 张纹理**（通过 Shader 多通道采样）
- **缩放（Scale）** 与 **旋转（Rotation）** 变换
- 基于 Unity **Job System** 的异步 Mesh 合并
- 自定义 **内存池**，减少 GC 压力
- **预制体（Prefab）** 系统，支持动态文字、图片替换

## 2. 背景与目标

### 2.1 问题背景

在游戏 HUD 场景中，大量小型 UI 元素（玩家名称、血条图标、状态图标等）如果各自独立渲染，会产生大量 Draw Call，严重影响帧率。Unity 内置 UGUI 的合批机制对于动态 UI（频繁位置/内容变更）效果有限。

### 2.2 核心目标

| 目标 | 说明 |
|------|------|
| 减少 Draw Call | 将 N 个 UI 元素合并为 1 次绘制调用 |
| 减少 GC 分配 | 通过内存池和共享缓冲区复用内存 |
| 支持异步处理 | 通过 Job System 将 Mesh 合并移出主线程 |
| 多纹理支持 | 单次 Draw Call 支持最多 8 张不同纹理 |
| 变换支持 | 支持 Scale、Rotation（通过 Matrix4x4） |
| 动态更新 | 运行时动态修改文字内容、图片、宽度 |

### 2.3 目标平台

- Unity 编辑器（Windows）
- 可部署至移动端（Android/iOS），对 Job System 和 Unsafe 代码有要求

## 4. 功能需求

### 4.1 核心渲染框架

**文件：** `Assets/Scripts/UIDataRender/Core.cs`

#### 需求 F-01：IUIDrawTarget 接口

- 所有 UI 组件（UIText、UIImage）必须实现 `IUIDrawTarget` 接口。
- 接口方法：`void DoGenerate(IUIData meshData, Transform root = null)`
- `DoGenerate` 负责将组件的顶点、UV、颜色、索引数据填充至 `IUIData` 对象。
- `root` 参数为可选的父级 Transform，用于计算相对变换矩阵。

#### 需求 F-02：IUIData 接口

- 所有 Mesh 数据容器（UIMeshData、UIMeshDataX）必须实现 `IUIData` 接口。
- 接口成员：
  - `int TextureIndex { get; set; }` — 纹理槽索引（1～7）
  - `void FillVertex(VertexHelper toFill, int flags)` — 从 VertexHelper 填充数据
  - `void UpdateTextureIndex(int index)` — 更新纹理索引（UV.z 通道）
  - `void TransformVertex(Matrix4x4 mtx)` — 变换顶点
  - `void FillToDrawData(...)` — 将数据追加至合并列表
  - `void FillToTriangleData(...)` — 仅追加索引数据（共享顶点缓冲模式）

---

### 4.2 几何内存管理

**文件：** `Assets/Scripts/UIDataRender/UIGeometry.cs`

#### 需求 F-03：共享几何缓冲区

- 维护全局共享的顶点（`vertex`）、绘制顶点（`drawVertex`）、UV（`uvs`）、颜色（`colors`）、索引（`indices`）数组。
- 初始容量：顶点 4096，索引 8192（`GROWTH = 4096`）。
- 数组不足时自动按 `GROWTH` 步长扩容（`Array.Resize`）。

#### 需求 F-04：内存分配（Alloc）

- `MeshSlim Alloc(int vertexCount, int indiceCount)`
- 从空闲链表（`openVertexList`、`openIndicesList`）中分配连续内存段。
- 优先查找精确匹配的空闲段；次之拆分更大的空闲段；无可用段时扩容数组。
- 返回 `MeshSlim` 描述符，记录偏移（Offset）和数量（Count）。

#### 需求 F-05：内存释放（Release / ReleaseSlice）

- `void Release(MeshSlim mesh)` — 释放顶点段和索引段。
- 释放后自动合并相邻空闲段（5 种边界情况：链表头前插、链表头合并、前邻合并、后邻合并、前后双向合并）。
- 合并规则：
  - Case 1：释放块位于所有空闲块之前 → 直接插入链表头
  - Case 2：释放块紧接链表头之前 → 合并入链表头
  - Case 3：释放块与前一节点相邻，同时与后一节点相邻 → 三段合一
  - Case 4：仅与前一节点相邻 → 合并入前节点
  - Case 5：仅与后一节点相邻 → 合并入后节点

#### 需求 F-06：内存重分配（ReAlloc）

- `void ReAlloc(int vertexCount, int indiceCount, ref MeshSlim mesh)` — 先释放旧段，再分配新段。
- 用于 UI 元素内容变化（如文字长度改变）后的大小调整。

### 4.3 Mesh 数据模型

**文件：** `Assets/Scripts/UIDataRender/UIMeshData.cs`、`UIMeshDataX.cs`

#### 需求 F-07：UIMeshData（独立数组模式）

- 每个实例持有独立的顶点、UV、颜色、索引数组。
- `uvs[]`：Vector4，XY 为 UV 坐标，Z 为纹理槽索引，W 为 flags。
- `CreateTextData(Text)` — 从 Unity Text 组件创建文字 Mesh 数据。
- `CreateImageData(Image)` — 从 Unity Image 组件创建图像 Mesh 数据（4 顶点矩形）。
- `FillToDrawData(...)` — 将顶点、UV、颜色、索引追加至合并缓冲区，位置加偏移 `localPosition`。
- `FillWithMatrix(...)` — 通过 `Matrix4x4` 变换后追加（支持 Scale/Rotation）。
- 不支持 Scale、Rotation、9 宫格（已在注释中标注）。

#### 需求 F-08：UIMeshDataX（共享缓冲区模式）

- 不持有独立数组，仅持有 `MeshSlim`（偏移 + 数量描述符）。
- 底层数据存储于全局静态 `UIGeometry geometry`。
- `FillVertex(VertexHelper, flags)` — 分配或重分配几何段，并将数据填入共享缓冲区的对应偏移位置。
- `FillToTriangleData(...)` — 将顶点加 `localPosition` 偏移写入 `drawVertex` 数组，并将索引写入输出列表。
- `Dispose()` — 释放持有的几何段。
- 内存效率更高，适合大量 UI 实例场景。

#### 需求 F-09：MeshSlim 描述符

- `struct MeshSlim`：`VertexOffset`、`VertexCount`、`IndicesOffset`、`IndicesCount`、`Index`（唯一标识，-1 表示无效）。
- 实现 `IEquatable<MeshSlim>`、`IEqualityComparer<MeshSlim>`。
- `Dispose()` — 将所有字段归零，`Index = -1`。

### 4.4 UI 组件系统

**文件：** `Assets/Scripts/UIDataRender/UI/`

#### 需求 F-10：UIText 组件

- 继承 Unity `Text` 组件，实现 `IUIDrawTarget`。
- `DoGenerate(IUIData meshData, Transform root)` — 通过 `VertexHelper` 生成文字顶点数据，将其写入 `meshData`。
- 支持 `UIOutline` 修饰符（描边）。
- 使用 `Utility.CalcMatrix(root, transform)` 计算相对于父节点的变换矩阵。
- 纹理索引固定为 1（字体纹理占用槽 1）。
- Flags 依 `UIOutline` 是否存在而变化（有描边时 flags=1，否则 flags=0）。

#### 需求 F-11：UIImage 组件

- 继承 Unity `Image` 组件，实现 `IUIDrawTarget`。
- `DoGenerate(IUIData meshData, Transform root)` — 通过 `VertexHelper` 生成图片顶点数据。
- 纹理索引由 `UIPrefabManager.GetTextureIndex(sprite.texture)` 动态获取。
- 支持父节点 Transform 层级变换。
- 运行时 Sprite 切换后通知 `UIPrefabManager` 更新纹理注册。

#### 需求 F-12：UIOutline 组件

- 继承 Unity `Outline` 组件。
- 重写 `ModifyMesh(VertexHelper toFill)` — 先记录原始顶点数量，调用 `base.ModifyMesh(toFill)` 添加描边顶点，再通知 `UIText` 填充两次（原始+描边）数据。
- 提供 `GetCount(VertexHelper)` — 返回原始顶点数量（描边前）。

### 4.5 预制体管理系统

**文件：** `Assets/Scripts/UIDataRender/UIPrefabManager.cs`、`UIPrefaHolder.cs`、`DataPrefabHolder.cs`、`UIPrefabOwner.cs`、`UIPrefabRegistration.cs`

#### 需求 F-13：UIPrefabOwner 组件

- MonoBehaviour，附加于 Prefab 根节点。
- 持有 `List<Transform> targets`，列出该 Prefab 下所有需要参与渲染的 UI 元素。

#### 需求 F-14：UIPrefaHolder 组件

- MonoBehaviour，负责 Prefab 实例的 Mesh 生命周期。
- `UseSlim(bool)` — 切换使用 `UIMeshDataX`（共享内存）或 `UIMeshData`（独立数组）。
- `BuildMesh(IUIDrawTarget[])` — 初始化数据持有者，向 `UIPrefabManager` 注册。
- `SetText(int index, string text)` — 动态更新指定索引的文字内容。
- `SetSprite(int index, Sprite spr)` — 动态更新指定索引的图片精灵。
- `SetWidth(int index, int width)` — 动态更新指定索引元素的宽度。
- `Fill(...)` — 将本实例的所有 Mesh 数据填充至渲染合并缓冲区。
- `OnDestroy()` — 从 `UIPrefabManager` 注销 dataHolder。

#### 需求 F-15：DataPrefabHolder\<T\>（泛型数据持有者）

- 泛型类，`T` 为 `UIMeshData` 或 `UIMeshDataX`。
- `BuildMesh(IUIDrawTarget[] draws)` — 为每个 draw target 创建 `T` 实例，并调用 `DoGenerate` 生成数据。
- `Fill(...)` — 遍历所有 `IUIData`，调用 `FillToDrawData` 或 `FillToTriangleData` 追加数据。
- `SetText/SetSprite/SetWidth/SetTextureIndex` — 转发动态更新请求。

#### 需求 F-16：UIPrefabRegistration

- 记录一个 Prefab 下所有 `IUIDrawTarget` 组件的引用数组。
- 构造时自动扫描 `UIPrefabOwner.targets`，为 `UIImage` 组件注册纹理。
- `Generate(IUIPrefabHolder holder)` — 触发 `holder.BuildMesh(draws)`。

#### 需求 F-17：UIPrefabManager（全局单例）

- 单例，全局管理所有 Prefab 的注册和纹理状态。
- `Register(IUIPrefabHolder)` — 为 Prefab 创建 `UIPrefabRegistration`，防止重复注册。
- `Generate(IUIPrefabHolder)` — 调用对应 Registration 的 `Generate`。
- `AddHolder / RemoveHolder` — 追踪所有活跃的 `IUIPrefabHolder`。
- `GetTextureIndex(Texture)` — 返回纹理在内部 `textures` 列表中的索引（1-based）。
- `UpdateTexture(Material)` — 将 `textures` 列表中的纹理赋值至 Material 的 `_MainTex1`～`_MainTex7`。
- **纹理注册**（`OnTextureRegister`）：添加新纹理至列表；若同一 GUID 的纹理变更，先注销旧纹理再注册新纹理。
- **纹理注销**（`OnTextureUnRegister`）：当某纹理被所有引用方移除后，从列表中移除；若被移除的不是最后一个，则将最后一个纹理换位至被删除位置（Swap-with-Last），并通知所有 Holder 更新 `TextureIndex`。

### 4.6 Job System 集成

**文件：** `Assets/OtherPlugins/managedTask/`、`Assets/Scripts/UIDataRender/MeshJobs.cs`、`MeshMergeJobs.cs`

#### 需求 F-18：IJobTask 与 ManagedCodeInJob

- `IJobTask` 接口：`void Execute()`，用于在 Job System 中执行托管代码。
- `ManagedCodeInJob`：将 `IJobTask` 包装为 Unity `IJob`，通过 `GCHandle` 固定托管对象，在 Job 中调用 `Execute()`。
- `LateUpdate` 中完成 Job（`JobHandle.Complete()`）并释放 `GCHandle`。

#### 需求 F-19：SharedArray\<T\>

- 同时提供 `T[]`（托管视图）和 `NativeArray<T>`（Job System 视图）。
- 通过 `GCHandle.Alloc(arr, GCHandleType.Pinned)` 固定托管数组，创建 NativeArray 指向同一内存。
- 隐式转换为 `T[]` 和 `NativeArray<T>`。
- `Dispose()` — 释放 NativeArray 和 GCHandle。

#### 需求 F-20：Mesh 合并 Job（MeshJobs）

- `MergeVertexJob`：将 N 个 UIMeshData 的顶点加位置偏移后合并至结果缓冲区。
- `MergeColorJob`：合并颜色数据（统一白色）。
- `MergeUVJob`：合并 UV 数据（使用 `UnsafeFastCopy.CopyVec4` 加速）。
- `MergeIndicsJob`：合并索引数据，并计算总顶点数和总索引数写入 `result_count`。

#### 需求 F-21：UnsafeFastCopy 工具

- `Copy<T>(T[] src, T* dst, int srcOffset, int dstOffset, int count)` — 泛型内存拷贝。
- `CopyVec4(Vector4[] src, Vector4* dst, ...)` — Vector4 数组专用拷贝。
- `CopyColor32(Color32[] src, Color32* dst, ...)` — Color32 数组专用拷贝。
- 底层使用 `UnsafeUtility.MemCpy`，绕过 C# 边界检查。

### 4.7 纹理管理

#### 需求 F-22：多纹理 Shader 支持

- Shader 属性：`_MainTex0`（字体纹理）、`_MainTex1`～`_MainTex7`（图片纹理）。
- UV 的 `z` 分量（0～7）决定采样哪张纹理。
- 单次 Draw Call 最多支持 **8 张纹理**。
- Atlas 支持：`IconAtlas.spriteatlas`、`MainAtlas.spriteatlas`（美术资源）。

#### 需求 F-23：RenderTexture 支持

- 提供 4x、16x、32x 缩放的 RenderTexture 配置（`RTexture4`、`RTexture16`、`RTexture32`）。
- 通过 `TestCopyRTexture.cs` 支持 RenderTexture 区域复制。

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 |
|------|------|
| Draw Call 数 | 理想情况下 ≤ 1（所有 HUD 元素合批） |
| 单帧 Mesh 合并耗时 | 通过 Job System 移至工作线程，主线程耗时 < 0.5ms |
| GC Alloc 每帧 | 目标为 0 bytes（复用内存池，避免 new） |
| 最大支持纹理数 | 8（Shader 硬限制） |
| 顶点池初始容量 | 4096 vertices，可按 4096 自动扩容 |
| 索引池初始容量 | 8192 indices，可按 4096 自动扩容 |

### 5.2 兼容性需求

- Unity 版本：2021.x 及以上（使用了 `VertexHelper.FillData2/FillData3` 等扩展）
- Burst Compiler：`TestUIGuiMetric` 中使用了 `[BurstCompile]`，需安装 Burst 包
- Jobs Package：`Unity.Jobs`、`Unity.Collections`，需安装对应包
